# Автоматизированная система обработки корпоративной почты

Система для классификации входящих обращений. Приложение читает входящие обращения из выбранной директории, классифицирует их по смысловым категориям с помощью ключевых слов и перемещает файлы в соответствующие папки, после чего формирует отчёт о проделанной работе (`stats.json`).

## Особенности
* **Динамические категории:** Категории и ключевые слова для классификации можно настраивать через файл `.env`.
* **Глобальный CLI:** Программа может быть установлена как консольная утилита `email-classifier`.
* **Многопоточность:** Обработка файлов может происходить параллельно.

## Требования
* Python 3.13+
* `pip` (стандартный пакетный менеджер Python)

## Установка и запуск

### Глобальная установка (CLI)
Для установки программы в качестве консольной утилиты выполните скрипт установки:
```bash
./install.sh
```

Вы также можете установить программу напрямую через `pip`:
```bash
pip install .
```

После этого программа будет доступна из любой директории по команде `email-classifier`. 
*Примечание: убедитесь, что директория бинарных файлов Python (например, `~/.local/bin` в Linux/macOS или `Scripts` в Windows) добавлена в вашу переменную окружения `PATH`.*

Пример использования:
```bash
email-classifier --input ./my_inbox --output ./processed_emails --jobs 4
```

### Локальный запуск
Для запуска скрипта без установки вы можете использовать локальный скрипт (предварительно убедитесь, что зависимости установлены, например, через `pip install -r pyproject.toml` или `pip install .`):
```bash
./run.sh -i inbox -o processed
```

### Аргументы командной строки
* `-i`, `--input` — Путь к папке с входящими письмами (по умолчанию: `inbox`).
* `-o`, `--output` — Путь к папке для отсортированных писем (по умолчанию: `processed`).
* `-j`, `--jobs` — Количество ядер CPU для параллельной обработки (по умолчанию: 4).
* `-h`, `--help` — Вызов справки.

## Настройка категорий (.env)
Система поддерживает динамическое добавление и изменение категорий через файл конфигурации `.env`. Программа ищет настройки в следующем порядке:
1. **Локальный файл `.env`** в директории, откуда запускается утилита.
2. **Глобальный файл `~/.email-classifier.env`** в домашней директории пользователя (автоматически создаётся при выполнении `install.sh`).

Пример файла конфигурации:
```env
# Список всех активных категорий (через запятую)
CATEGORIES="spam,alerts,urgent,newsletters,hr_documents,invoices"

# Ключевые слова для встроенных категорий (переопределение)
SPAM_KEYWORDS="casino,win,discount,скидк,выигрыш,реклама,казино"
ALERTS_KEYWORDS="alert,grafana,zabbix,prometheus,noreply,daemon"
URGENT_KEYWORDS="urgent,срочно,критичн,падает,ошибка 500,инцидент,недоступен"
NEWSLETTERS_KEYWORDS="дайджест,newsletter,рассылка,новост"
HR_DOCUMENTS_KEYWORDS="отпуск,больничный,инструкция,согласование,договор,заявление"

# Ключевые слова для ваших пользовательских категорий
INVOICES_KEYWORDS="счет,invoice,оплата,payment,bill"
```

Если письмо не попадает ни под одну из категорий, оно будет помещено в папку `unknown`. Письма с ошибками чтения или повреждёнными данными попадают в папку `errors`.

## Тестирование
Код покрыт юнит-тестами с использованием фреймворка `pytest`. Тесты автоматически адаптируются под категории по умолчанию.

Убедитесь, что установлен фреймворк `pytest`, а затем выполните:
```bash
pytest tests/ -v
```

## Архитектура
* `structures.py` — класс `Letter`, который отвечает за парсинг текста письма, обработку заголовков и извлечение метаинформации (дата, отправитель, тема), а также `NamedTuple` класс `Category` для инкапсуляции имени категории и списка её ключевых слов.
* `classifier.py` — основная логика работы, парсинг конфигурации `.env`, вызов классификации и работа с файловой системой.
* `install.sh` / `run.sh` — bash-скрипты для глобальной установки и локального запуска.
* `tests/` — модульные тесты для проверки корректности присвоения категорий.
