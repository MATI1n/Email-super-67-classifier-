# Автоматизированная система обработки корпоративной почты

Репозиторий содержит решение хакатона по автоматизации обработки входящих писем службы поддержки. Проект разделен на две основные части:

1. **Базовое решение (CLI):** Консольное приложение для классификации писем по ключевым словам. Находится в папке [cli/](file:///Users/egormashtakov/PycharmProjects/Email-super-67-classifier-/cli).
2. **Продвинутое решение (MailPilot):** Веб-приложение с графическим интерфейсом на React (Vite) и бэкендом на FastAPI, объединяющее интеллектуальную суммаризацию писем и умный гибридный поиск. Находится в папке [mailpilot/](file:///Users/egormashtakov/PycharmProjects/Email-super-67-classifier-/mailpilot).

---

## Структура проекта

* [cli/](file:///Users/egormashtakov/PycharmProjects/Email-super-67-classifier-/cli) — Базовое решение:
  * `classifier.py` — Логика классификации писем по правилам.
  * `structures.py` — Описание структур данных (`Letter`, `Category`).
  * `install.sh` / `run.sh` — Скрипты установки и запуска CLI.
  * `tests/` — Модульные тесты для CLI-классификатора.
  * `README.md` — Инструкция к базовому решению.
* [mailpilot/](file:///Users/egormashtakov/PycharmProjects/Email-super-67-classifier-/mailpilot) — Продвинутое решение:
  * `backend/` — Микросервисы на FastAPI, расширенные интеллектуальными функциями:
    * `app/ai.py` — Суммаризатор писем, использующий API OpenRouter (модель LLM) с разбором структурированных ответов.
    * `app/search.py` — Умный поиск по письмам, основанный на векторной БД Qdrant (эмбеддинги Jina v5) с извлечением сущностей (отправителя) через LLM.
  * `frontend/` — Клиентская часть на React + Vite.
  * `run.sh` — Скрипт автоматической сборки фронтенда, настройки окружения и запуска веб-приложения.

## Инструкция по запуску

### 1. Базовое решение (CLI)

Перейдите в папку CLI и запустите локальный скрипт:
```bash
cd cli
./run.sh -i tests/data -o processed
```

Для глобальной установки CLI:
```bash
cd cli
./install.sh
# Теперь утилита доступна из любого места:
email-classifier --help
```

### 2. Продвинутое решение (MailPilot с ИИ-сервисами)

Для работы интеллектуальных функций (суммаризации и поиска) задайте ключи в файле `.env` в директории `mailpilot/backend/.env`.

Пример конфигурации (`.env`):
```env
# --- AI-суммаризатор и извлечение метаданных (OpenRouter) ---
OPENROUTER_API_KEY_2=your_openrouter_api_key
BASE_MODEL_URL=google/gemini-2.5-flash # или любая другая модель

# --- Семантический поиск (Qdrant) ---
QDRANT_API_KEY=your_qdrant_api_key
```

Запуск веб-приложения (автоматически развернет виртуальное окружение, установит зависимости и соберет React-клиент):
```bash
cd mailpilot
./run.sh
```

После запуска интерфейс будет доступен по адресу: **http://localhost:8000**

## Интеграция интеллектуальных микросервисов (Часть 3)

В бэкенд MailPilot были интегрированы продвинутые возможности из Части 3:
1. **Интеллектуальная суммаризация писем:** Эндпоинт `/api/emails/{mail_id}/summarize` использует LLM через API OpenRouter. Система формирует структурированную сводку (суть проблемы, ПО, приоритет, настроение пользователя) на языке оригинала (по умолчанию на русском).
2. **Умный поиск:** Эндпоинт `/api/search` выполняет поиск по векторной БД Qdrant. Запросы проходят через LLM для автоматического выделения имени или email отправителя, накладывая соответствующие жесткие фильтры на результаты векторного поиска на базе модели `jinaai/jina-embeddings-v5-omni-nano`.
